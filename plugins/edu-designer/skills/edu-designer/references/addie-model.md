# ADDIE 모델 — 교수설계의 표준 프레임워크

## 개요
ADDIE는 가장 널리 사용되는 체제적 교수설계 모델로, 5단계 순환 프로세스를 따른다.
체계적이고 검증된 교육 과정을 설계할 때 기본 모델로 적용한다.

## 5단계

### 1. Analysis (분석)
- **목적**: 교육 니즈, 학습자 특성, 환경 분석
- **핵심 활동**:
  - 수행 격차 분석 (현재 수준 vs 목표 수준)
  - 학습자 분석 (사전 지식, 동기, 학습 스타일)
  - 과제 분석 (학습해야 할 지식/기술/태도)
  - 환경 분석 (학습 환경, 제약 조건)
- **산출물**: 요구 분석 보고서, 학습자 프로파일

### 2. Design (설계)
- **목적**: 학습목표 수립, 구조 설계, 평가 전략 수립
- **핵심 활동**:
  - 수행목표 진술 (Bloom 분류학 기반)
  - 평가 도구 설계 (목표와 정렬)
  - 교수 전략 선택
  - 콘텐츠 계열화 (순서 결정)
  - 매체 선정
- **산출물**: 교수설계 문서, 스토리보드, 프로토타입

### 3. Development (개발)
- **목적**: 설계를 기반으로 교육 자료 제작
- **핵심 활동**:
  - 교수 자료 개발 (교재, 슬라이드, 영상 등)
  - 학습 활동 개발
  - 평가 문항 개발
  - 파일럿 테스트
- **산출물**: 교육 콘텐츠, 교수자 가이드, 학습자 자료

### 4. Implementation (실행)
- **목적**: 설계된 교육 실시
- **핵심 활동**:
  - 교수자 교육 (Train-the-Trainer)
  - 학습 환경 준비
  - 교육 실시
  - 학습 촉진 및 지원
- **산출물**: 교육 실시 기록, 참여자 피드백

### 5. Evaluation (평가)
- **목적**: 교육 효과성 측정 및 개선
- **핵심 활동**:
  - 형성평가 (과정 중 개선 목적)
  - 총괄평가 (과정 후 효과 측정)
  - Kirkpatrick 4단계 평가:
    - L1 반응: 만족도
    - L2 학습: 지식/기술 습득
    - L3 행동: 현업 적용
    - L4 결과: 조직 성과 기여
- **산출물**: 평가 보고서, 개선 권고사항

## 적용 가이드

### 언제 ADDIE를 쓰나
- 체계적이고 검증된 교육 과정이 필요할 때 (기본값)
- 이해관계자가 많고 승인 프로세스가 필요한 조직 교육
- 반복 운영되는 정규 교육 과정
- 교육 효과를 정량적으로 측정해야 할 때

### 언제 다른 모델을 고려하나
- 개발 기간이 2주 미만으로 촉박할 때 → SAM/래피드 프로토타이핑
- 문제 해결 중심의 실무 교육 → 메릴의 교수원리
- 복잡한 다단계 교육 시스템 → 딕앤케리 모델

### 커리큘럼 설계 시 ADDIE 적용 포인트
- Step 1(니즈 분석 인터뷰)은 Analysis 단계에 해당
- Step 2(모델 선택 + 설계)와 Step 3(요약 테이블)은 Design 단계에 해당
- Step 4(설계서 저장)는 Design→Development 경계 — 이후 교안 제작이 Development
- 설계서에 Evaluation 전략(평가 계획)을 반드시 포함 (평가 없는 교육은 의미 없음)
