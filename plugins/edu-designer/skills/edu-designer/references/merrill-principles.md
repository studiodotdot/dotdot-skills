# 메릴의 교수원리 (Merrill's First Principles of Instruction)

## 개요
David Merrill이 주요 교수설계 이론들의 공통 원리를 추출하여 정리한 5가지 핵심 교수원리.
"어떤 이론을 쓰든 이 원리가 적용되면 학습 효과가 높아진다"는 것이 핵심 주장.
특히 문제 해결 중심의 실무 교육에 강력하다.

## 5가지 원리

### 1. 문제 중심 (Problem-Centered)
- 학습은 실제적인 문제나 과제를 중심으로 조직되어야 한다
- 학습자가 "이걸 왜 배우는지"를 처음부터 알아야 한다
- **적용**: 교육 시작 시 실제 업무 상황/문제를 먼저 제시
- **예시**: "불만 고객이 전화를 끊겠다고 할 때 어떻게 하시겠습니까?" → 이 문제를 해결하는 것이 이번 모듈의 목표

### 2. 활성화 (Activation)
- 학습자의 기존 지식과 경험을 활성화해야 한다
- 새로운 지식을 기존 지식 구조에 연결
- **적용**: "여러분이 겪은 비슷한 상황을 떠올려보세요" (경험 회상)
- 선수지식이 부족하면 보충 활동 제공
- **예시**: 모듈 도입부에서 사전 경험 공유, 사전 퀴즈, 브레인스토밍

### 3. 시연 (Demonstration)
- 새로운 지식/기술을 "보여줘야" 한다 (단순 설명이 아님)
- Tell me ≠ Show me
- **적용**: 개념 → 구체적 예시, 절차 → 시범(데모), 원리 → 적용 사례
- 정확한 시연 + 오류 시연 → 비교를 통한 학습
- **예시**: 모범 응대 영상, Before/After 비교, 전문가 시범

### 4. 적용 (Application)
- 학습자가 직접 해봐야 한다 (연습 + 피드백)
- 점진적 난이도 증가 (scaffolding → fading)
- **적용**: 롤플레이, 사례 분석, 시뮬레이션, 프로젝트
- 즉각적이고 구체적인 피드백 제공
- **예시**: 모의 상담 실습, 사례 기반 의사결정 연습

### 5. 통합 (Integration)
- 배운 것을 자신의 일상/업무에 통합해야 한다
- "이걸 내 업무에서 어떻게 쓸 수 있을까?"
- **적용**: 현업 적용 계획 수립, 동료와 공유, 성찰 활동
- **예시**: 액션 플랜 작성, 현업 적용 과제, 학습 일지

## 적용 가이드

### 언제 메릴의 교수원리를 쓰나
- 실무 적용이 핵심인 직무 교육 (CS, 영업, 리더십 등)
- 문제 해결 능력을 길러야 하는 과정
- "알기만 하면 안 되고 할 줄 알아야 하는" 교육
- 기존 과정이 이론 위주로 편향된 경우 리뉴얼할 때

### 커리큘럼 설계 시 적용 포인트
- 각 모듈의 내부 구조를 "문제→활성화→시연→적용→통합" 순서로 설계
- ADDIE/딕앤케리의 큰 틀 안에서 각 차시의 교수 전략으로 활용
- 특히 "실습 비중 높여줘"라는 요구 시 메릴 원리 강화
- 각 모듈별 도입 문제(현실 시나리오)를 반드시 포함

### 모듈 내부 시간 배분 가이드 (메릴 기반)
| 단계 | 비중 | 활동 |
|------|------|------|
| 문제 제시 | 10% | 실제 사례/상황 제시 |
| 활성화 | 10% | 경험 공유, 사전 지식 점검 |
| 시연 | 25% | 개념 설명 + 예시/시범 |
| 적용 | 40% | 실습, 롤플레이, 연습 |
| 통합 | 15% | 정리, 현업 적용 계획 |
