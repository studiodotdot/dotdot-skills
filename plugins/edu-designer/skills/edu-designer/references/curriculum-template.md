# 커리큘럼 출력 포맷 템플릿

## 개요
edu-designer 스킬의 최종 출력 형식. 이 템플릿에 맞춰 일관된 커리큘럼 설계서를 생성한다.

## 출력 구조

```markdown
# {교육 과정명} 커리큘럼 설계서

## 과정 개요

| 항목 | 내용 |
|------|------|
| 교육 과정명 | {과정명} |
| 교육 목적 | {1-2문장} |
| 대상 학습자 | {직군, 직급, 인원} |
| 총 교육 시간 | {시간} |
| 교육 형태 | {오프라인/온라인/블렌디드} |
| 적용 교육 모델 | {ADDIE/딕앤케리/메릴} |

## 학습목표

이 과정을 이수한 학습자는 다음을 할 수 있다:
1. {Bloom 행동 동사 기반 목표 1}
2. {Bloom 행동 동사 기반 목표 2}
3. {Bloom 행동 동사 기반 목표 3}

## 모듈 구조도

{모듈 간 관계를 텍스트로 시각화}

모듈 1: {제목} ──→ 모듈 2: {제목} ──→ 모듈 3: {제목}
     │                    │
     └── 선수: 없음        └── 선수: 모듈 1
                                연계: 모듈 1에서 배운 {X}를 실전 적용

## 상세 커리큘럼

### 모듈 {N}: {모듈명}

| 항목 | 내용 |
|------|------|
| 학습목표 | {Bloom 행동 동사 기반} |
| 핵심 내용 | • {주제 1}<br>• {주제 2}<br>• {주제 3} |
| 교수방법 | {방법} — {선택 이유} |
| 소요시간 | {분} |
| 평가방법 | {형성/총괄/실기 등} |
| 선수 모듈 | {없음 또는 모듈 N} |
| 연계 포인트 | {이전/이후 모듈과의 관계} |

#### 차시 구성 (메릴 교수원리 기반)

| 단계 | 시간 | 활동 |
|------|------|------|
| 문제 제시 | {분} | {실제 상황/사례 제시} |
| 활성화 | {분} | {사전 경험 공유, 퀴즈} |
| 시연 | {분} | {개념 설명, 예시, 시범} |
| 적용 | {분} | {실습, 롤플레이, 연습} |
| 통합 | {분} | {정리, 현업 적용 계획} |

{모듈 수만큼 반복}

## 설계 근거

### 적용 교육 모델

{모듈 순서를 왜 이렇게 배치했는지}
- 구조화: {기본→심화, 개념→실무 등}
- 계열화: {Easy→Hard, Concrete→Abstract, Known→Unknown}

### 교수방법 선택 이유
{각 모듈에서 해당 교수방법을 선택한 근거}

## 평가 계획

| 평가 유형 | 시점 | 대상 모듈 | 방법 | 기준 |
|----------|------|----------|------|------|

| 모듈 | 이론 | 실습 | 평가 | 합계 |

| **합계** | **{분}** | **{분}** | **{분}** | **{분}** |
```

## 포맷 규칙

1. 모든 학습목표는 Bloom 행동 동사로 시작
2. 교수방법에는 반드시 선택 이유 포함
3. 모듈 간 연계 포인트 명시 (고립된 모듈 없어야 함)
4. 시간 합계가 총 교육 시간과 일치해야 함
5. 2시간 초과 과정은 휴식을 배치하고 시간 배분 요약에 포함 (휴식 포함 여부를 과정 개요에 명시)
6. 평가 계획은 학습목표와 정렬(alignment)
7. 설계 근거 섹션은 내부 보고/결재용으로 반드시 포함
8. 파일명은 `{교육과정명}_커리큘럼_{YYMMDD}.md`
